Run the following recipe in order. If there are any failures not
handled by this recipe, stop there and report failure. If all steps
succeed, report success.

1. Merge the `master` branch from
   https://github.com/project-everest/everparse into this branch.

   If there are conflicts on `opt/hashes.Makefile`, ignore the
   incoming changes and keep ours.

   If there are other conflicts, stop here and report failure.

2. Run `make -C opt advance`

3. If steps 1 and 2 succeeded with no changes, stop here and report
   success.

4. Commit the changes in opt/hashes.Makefile

5. Run `make clean`

6. Run `make -j16 cbor-snapshot` with the following guidelines:

   * When running `make` or F*, collect its output to a logfile and
     grep from the logfile to diagnose errors.

   * If there are any F* failures, fix the .fst and .fsti files using
     agents and skills from `opt/FStar/.github/agents` and
     `opt/FStar/.github/skills`, and try again until proofs
     succeed. Do not insert and leave any admit()/assume behind (other
     than FStar.SizeT.fits_u64 in Pulse code.)

   * If there are any failures in handwritten C or Rust code, fix
     them, and try again until compilation succeeds.

7. Run `make -j16 cose-snapshot` and follow the same guidelines as in
   Step 6.

8. Run `make -j16 test` and follow the same guidelines as in
   Step 6.

9. Commit the changes other than the files generated by
   `cbor-snapshot` or `cose-snapshot`

10. Commit the files generated by `cbor-snapshot` or `cose-snapshot`

11. Push.
